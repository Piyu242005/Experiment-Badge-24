<details>   <summary>🌐 GitHub Pages Supported Technologies</summary>

### ✅ Supported Languages & Technologies

* 🌐 **HTML5** – Website structure
* 🎨 **CSS3** – Styling and layouts
* ⚡ **JavaScript (ES6+)** – Interactivity and dynamic content
* ⚛️ **React** (Static Build)
* 🟢 **Vue.js** (Static Build)
* 🅰️ **Angular** (Static Build)
* 🔷 **TypeScript** (Compiled to JavaScript)
* 🖼️ **Images** (PNG, JPG, SVG, WebP)
* 📄 **Markdown (.md)**
* 🎯 **Jekyll** Static Sites

### ❌ Not Supported Directly

* 🐍 Python
* 🚀 Streamlit
* ⚡ FastAPI
* 🌶️ Flask
* 🎸 Django
* 🟩 Node.js Backend
* 🗄️ Databases (MySQL, PostgreSQL, MongoDB)

### 💡 Why?

GitHub Pages is a **static hosting service**. It can only serve pre-built files such as HTML, CSS, JavaScript, images, and other static assets. It cannot run backend servers, Python code, APIs, or databases.

### 🚀 Best Use Cases

* Personal Portfolio Websites
* Documentation Sites
* Project Landing Pages
* Blogs
* Resume Websites
* React/Vue Static Applications

### 🤖 For AI/ML Projects

Use:

* GitHub → Source Code
* Streamlit Cloud → Live Demo
* GitHub Pages → Project Landing Page

This combination provides the best developer and recruiter experience.
