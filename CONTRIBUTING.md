**What it is**
`CONTRIBUTING.md` is a guide that explains **how others can contribute** to a project correctly.

**Main Purpose**

* Maintain code quality.
* Reduce mistakes and confusion.
* Standardize contributions.

**Typical Sections**

1. **How to Start**

   * Fork the repository.
   * Clone it locally.
   * Create a new branch.

2. **Development Setup**

   * Required tools (Python version, libraries).
   * Installation steps.
   * Environment setup.

3. **Coding Standards**

   * Code style rules (PEP8, naming).
   * File structure.
   * Commenting practices.

4. **Commit Rules**

   * Clear commit messages.
   * One change per commit.

5. **Pull Request (PR) Process**

   * How to create a PR.
   * Required checks/tests.
   * Review and approval steps.

6. **Testing Guidelines**

   * How to run tests.
   * Test coverage expectations.

7. **Issue Reporting**

   * Bug report format.
   * Feature request rules.

8. **Code of Conduct**

   * Expected behavior.
   * Respectful communication.

**Why it is Important**

* Helps new contributors contribute faster.
* Saves maintainer time.
* Keeps the project professional and scalable.

**In short:**
`CONTRIBUTING.md` is the **rulebook for contributors** to work efficiently and correctly.
